## What Happens When an AI Works in This Repo?

When Claude (or another AI) works on tasks in this repository, it operates within
a structured system of behavioral instructions. These instructions define:

- **What role the AI plays** (implementer, reviewer, planner, etc.)
- **What it's allowed to do** (write files, run tests, commit code)
- **What it must not do** (skip tests, merge without review, etc.)
- **How it makes decisions** (which agent to invoke, when to ask for help)
- **How it communicates** (structured output formats, changelog discipline)

All of these instructions are visible in the `.claude/` directory and mirrored here.

---

## The Agent Roster

### 🔨 Implementer
The implementer agent is responsible for **writing code and making changes**.
It follows plans generated by the planner and stays within defined scope.
It is instructed to write tests alongside implementations and to use
structured commit messages.

### ⚖️ Judge
The judge agent **evaluates outputs** for quality, correctness, and alignment
with requirements. It provides structured verdicts and can block progress
if quality standards aren't met.

### 📋 Planner
The planner agent **breaks down complex tasks** into step-by-step implementation
plans. It considers dependencies, risks, and sequencing before any code is written.

### 🔍 Reviewer
The reviewer agent **examines completed work** for issues, code quality,
and adherence to project standards. It acts as a final gate before changes
are considered done.

### 🔭 Scout
The scout agent **gathers context and does research** before implementation begins.
It reads existing code, checks documentation, and surfaces relevant information
so other agents can work effectively.

## Guardrails and Limits

The following guardrails are built into the agent system:

| Guardrail | How It's Enforced |
|-----------|------------------|
| Tests must pass before commit | `auto-fix` and `test-first-executor` skills |
| Changelog must be updated | `changelog-enforcer` skill |
| Scope must not drift | `scope-guard` skill |
| Risky code gets extra review | `risky-module-review` skill |
| Dependencies are audited | `dependency-audit` skill |
| Destructive ops are sandboxed | `sandboxed-exec` skill |

---

## Human Oversight Points

The agent system is designed with explicit human oversight at:

1. **Issue creation** — a human defines what work should be done
2. **Plan review** — plans can be reviewed before implementation starts
3. **PR review** — all changes go through standard GitHub PR review
4. **Release gate** — `release-readiness` skill requires explicit sign-off

---

## What the AI Won't Do

Based on the current skill and agent definitions, the agents in this repo are
NOT instructed to:

- Autonomously deploy to production
- Modify security-sensitive configurations without flagging
- Merge pull requests without review
- Suppress test failures
- Hide errors or unexpected behaviors from the human operator
